# 공정 규격 작성 가이드

## 핵심 원칙

구매규격서의 "Ⅲ. 성능 및 규격" 섹션은 **특정 제품을 명시하지 않으면서 실질적으로 원하는 제품만 충족하도록** 작성해야 한다. 동시에, 공정 거래 관점에서 합리적 기술 근거가 있어야 한다.

## 금지사항

1. **브랜드명, 모델명 기재 불가** — "Intel Xeon", "NVIDIA A100", "AMD EPYC" 등 직접 언급 금지
2. **독점기술명 기재 불가** — "Infinity Fabric", "CUDA Core" 등 특정 업체 기술명 금지
3. **과도하게 좁은 범위** — "정확히 64코어" 대신 "64코어 이상" 사용
4. **성능과 무관한 물리적 치수** — 섀시 깊이, PCB 층수 등으로 필터링 금지
5. **검증 불가능한 사양** — 내부 설계, 부품 공급사 등 공개되지 않는 사양 금지

## 규격 수치 설정 방법

### 기본 공식
```
규격값 = 타겟 모델 실제값 × 0.90~1.0 (항목 특성에 따라 조절)
```

- 이산적 수치 (코어 수, 슬롯 수): 타겟 모델과 동일하거나 한 단계 아래
- 연속적 수치 (클럭, 대역폭): 타겟 모델의 90~100%
- 인증/등급: 타겟 모델과 동일 등급

### 예시

| 타겟 모델 실제 사양 | 규격서 기재 | 차별화 근거 |
|---|---|---|
| CPU 64코어/128쓰레드 | 소켓당 64코어/128쓰레드 이상 | 경쟁사 대부분 소켓당 최대 56코어 |
| L2 64MiB, L3 256MiB | 코어당 L2 1MiB 이상, L3 256MiB 이상 | 경쟁사 L3 통상 128MiB 이하 |
| DDR5-4800 24슬롯 6TB max | DDR5-4800 이상, 12채널, 24슬롯 이상, 6TB 확장 | 채널 수+슬롯 수 조합으로 차별화 |
| PCIe 5.0 x16 4개 | PCIe 5.0 x16 풀하이트 4개 이상 | GPU 가속 요구사항으로 정당화 |
| 12 NVMe 핫스왑 베이 | NVMe 핫스왑 베이 8개 이상 | 여유 있게 설정하되 경쟁사 4개짜리 탈락 |
| 80 PLUS Titanium 2200W | 80 PLUS Titanium 이상, 2200W 이상/PSU | 인증 등급+용량 조합 |
| IPMI 2.0 + KVM-over-IP | IPMI 2.0 호환, 원격 KVM-over-IP 지원 | 관리 기능 조합으로 필터링 |

### 공정한 차별화 전략

**핵심**: 각 스펙 항목은 개별적으로는 합리적이어야 하지만, **전체 조합**을 동시에 만족하는 경쟁 제품이 매우 제한적이도록 설계한다.

#### 좋은 차별화 (합리적 기술 근거 있음)
- "소켓당 64코어/128쓰레드 이상" → 대규모 병렬 연산에 필요한 코어 수
- "L3 캐시 256MiB 이상" → 대용량 데이터셋의 메모리 계층 최적화에 필요
- "PCIe 5.0 x16 4개 이상" → GPU 4장 동시 장착으로 딥러닝/MD 가속
- "ECC RDIMM 12채널 이상" → 과학 연산의 데이터 무결성 및 대역폭
- "80 PLUS Titanium" → 대규모 GPU 부하 시 전력 효율과 발열 관리
- "NVMe 핫스왑 베이 8개 이상" → 수십 TB 시퀀싱 데이터의 고속 I/O

#### 나쁜 차별화 (근거 불충분 또는 편파적)
- "CPU 기본 클럭 정확히 2.45GHz" → 특정 모델만 해당하는 정확한 수치
- "섀시 깊이 830mm 이하" → 성능과 무관한 물리적 제한
- "BMC 칩셋 AST2600" → 특정 부품 지정
- "전원 커넥터 16핀 x 4" → 내부 설계 세부사항
- "팬 모듈 정확히 6개" → 냉각 성능이 아닌 구조적 제한

## 규격 항목 작성 요령

### 항목 수와 상세도

- **최소 8개 이상, 10개 내외**가 적절
- 각 항목은 **한 줄로 해당 카테고리의 핵심 요구사항을 모두 포함**해야 함
- 너무 적으면 (5개 이하) 경쟁 제품이 쉽게 충족
- 템플릿은 8개 슬롯이 기본이나, 스크립트가 자동으로 추가 슬롯을 생성함

### 항목별 서술 구조

각 항목은 다음 구조로 작성한다:
```
[카테고리]: [기술 요구사항 1], [기술 요구사항 2], [수치 요건], [부가 요건]
```

좋은 예:
```
메모리: DDR5-4800 이상 ECC RDIMM 지원, 소켓당 12채널 이상 메모리 인터페이스, 최소 512GB 장착(소켓당 256GB), 최대 6TB 이상 확장 가능, 슬롯 24개 이상
```

나쁜 예 (너무 짧음):
```
메모리: DDR5 512GB 이상
```

## 기타 조건(Remarks) 작성 요령

**최소 5개 이상** 기재하며, 다음 카테고리를 고려한다:

| 카테고리 | 내용 | 예시 |
|---|---|---|
| 설치 | 설치 장소, 설치 주체, 설치 범위 | 서버랙 장착, 전원/네트워크 연결 포함 |
| 보증 | 보증 기간, A/S 응답 시간, 부품 확보 | 3년 무상보증, 48시간 현장 수리 |
| 납기 | 납품 기한, 지연 시 조치 | 계약 후 60일 이내, 지연 시 2주 전 통보 |
| 검증 | 납품 시 테스트, 결과 보고 | CPU/메모리/SSD 스트레스 테스트 |
| 교육 | 운영 교육, 기술 지원 창구 | 기본 운영 교육 1회, 기술 지원 연락처 |
| 문서 | 매뉴얼, 보증서 | 설치/유지보수 가이드, 보증서 사본 |
| 호환성 | OS 지원, 소프트웨어 호환성 | Ubuntu 22.04 LTS 정상 동작 확인 |

## 장비 유형별 규격 패턴

### 서버/워크스테이션
핵심 항목: CPU (코어/쓰레드, 클럭, 캐시, TDP), 메모리 (타입/채널/슬롯/최대용량), 스토리지 (인터페이스/베이수/핫스왑/RAID), 네트워크 (속도/포트수/관리포트), PSU (효율등급/용량/이중화), GPU 확장성 (슬롯수/대역폭/전력), 관리기능 (BMC/IPMI/원격관리), 폼팩터/인증

### 시퀀서 (DNA/RNA)
핵심 항목: 시퀀싱 방식 (롱리드/숏리드), 최대 출력량 (Gb/run), 최대 리드길이 (bp/kb), 원시 정확도 (%), 동시 처리 셀 수, 플로우셀 종류, 데이터 출력 형식, 런타임, 시약 소모량, 소프트웨어 통합

### 현미경
핵심 항목: 광학 방식 (공초점/광시야/초해상도), 배율 범위, XY/Z 해상도 (nm), 검출기 타입 (PMT/HyD/sCMOS), 레이저 파장 수/종류, 스캔 속도, 이미징 모드, 인큐베이션 호환, 자동화 스테이지, 소프트웨어

### 원심분리기
핵심 항목: 최대 RPM/RCF, 로터 호환성 (고정각/스윙/수직), 온도 제어 범위, 최대 용량, 가감속 프로파일, 프로그래밍, 소음 수준, 안전 기능 (로터 인식/이중 잠금)

### 초저온냉동고 (-80°C)
핵심 항목: 최저 온도, 보관 용량 (L/박스 수), 복원 시간, 전력 소비, 냉매 종류, 알람/모니터링, 도어 구조, 비상 배터리

## "이상/이하" 표현 가이드

| 상황 | 표현 |
|---|---|
| 성능이 높을수록 좋음 | "XX 이상" |
| 정밀도(오차)가 낮을수록 좋음 | "XX 이하" |
| 범위가 넓을수록 좋음 | "최소 XX ~ 최대 YY" |
| 소비가 적을수록 좋음 | "XX 이내" |
| 특정 기능 유무 | "XX 지원" 또는 "XX 기능 포함" |
| 인증/규격 | "XX 인증" 또는 "XX 호환" |
| 등급 이상 | "XX 이상 인증" (예: "80 PLUS Titanium 이상") |
